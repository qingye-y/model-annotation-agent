# 商品审核大模型质检标注工作台 - 项目介绍

> 更新日期：2026-05-12

---

## 一、项目定位

这是一个面向**商品审核大模型质检**的全流程标注工作台，覆盖从数据拉取、机审任务管理、人工标注、质检修正到 Badcase 管理的完整闭环。

采用 **纯静态 HTML 前端 + Flask 后端** 的架构，前端通过 iframe 方式加载各子页面。

---

## 二、前端部分（9 个页面 + 主框架）

### 主框架页

| 文件 | 功能 |
|------|------|
| `index.html` | 主框架，左侧导航栏（9个菜单）+ 顶部用户信息栏 + 右侧 iframe 内容区。支持侧边栏收起/展开，状态持久化到 localStorage |

### 子页面（通过 iframe 加载）

| 页面 | 功能描述 |
|------|----------|
| **dashboard.html** | 数据看板 — 展示总数据量、合规/违规分布、准确率、不一致率等核心指标，含 ApexCharts 图表 |
| **model_task.html** | 机审任务中心 — 实例筛选（5个实例）、规则筛选、时间范围、分页数据展示、状态监控 |
| **dispatch_center.html** | 任务调度中心 — Tab 切换（规则调度/实例调度）、规则卡片管理、调度概览统计 |
| **annotation_list.html** | 标注任务 — 核心标注页面，支持批量分配、标注状态筛选、商品详情弹窗、标注结果提交 |
| **qc_center.html** | 质检修正中心 — 质检概览统计、抽检配置、质检任务列表、修正操作 |
| **badcase_center.html** | Badcase 管理中心 — Badcase 筛选（按实例/规则/时间/状态）、统计卡片、数据表格、详情查看 |
| **get_log.html** | 获取日志 — 数据拉取日志查询、批次详情、日志导出 |
| **account_management.html** | 账号管理 — 标注员账号的增删改查、权限分配、密码重置、日配额管理 |
| **settings.html** | 系统配置 — 标注标签管理、提示词规则配置、实例管理、规则与实例绑定关系 |

### 前端基础设施

| 文件 | 作用 |
|------|------|
| `static/common.css` | 全局样式系统，定义了 CSS 变量（主色、功能色、布局、圆角、阴影等）和基础组件样式 |
| `static/config.js` | 前端配置中心，管理标注标签、提示词规则、实例列表、规则-实例绑定关系、标注员用户数据，全部持久化到 localStorage |

## 三、后端部分（Flask + SQLite）

### 技术栈

- **Flask 2.3.3** + **Flask-SQLAlchemy** + **Flask-Login** + **PyJWT** + **requests**

### 数据库模型（5 张表）

| 表名 | 作用 |
|------|------|
| `user` | 用户账号 — 用户名、密码哈希、角色(admin/annotator)、姓名、日配额、是否激活 |
| `raw_data` | 原始商品数据 — 存储从 iData 拉取的所有商品审核字段（供应商ID、AI审核结果、人审结果、拒绝原因、商品图片链接、商品链接、实例编码等 30+ 字段） |
| `fetch_log` | 拉取日志 — 记录每次数据拉取的批次号、环境、实例、抽样比例、拉取数量、合规/违规统计 |
| `sql_template` | SQL 模板 — 存储可配置的 SQL 查询模板，支持动态参数替换 |
| `annotation` | 标注记录 — 关联 raw_data 和 user，记录标注结果、错误标签、备注、提交状态 |

### 蓝图（4 个模块）

| 蓝图 | 路由前缀 | 功能 |
|------|----------|------|
| `auth.py` | `/api/auth` | 登录页面 `/login`、API 登录 `/api/auth/login`（返回 JWT Token）、登出、当前用户信息、用户管理 CRUD（增删改查）、密码重置 |
| `data_fetch.py` | `/api/data-fetch` | **核心数据拉取逻辑** — 动态构建 SQL、分页查询 iData、随机抽样、数据入库、记录日志、触发模型 B 互检（Mock） |
| `dashboard.py` | `/api` | 统计 API：总览 `/overview`、按实例 `/by-instance`、按用户 `/by-user`、趋势 `/trend`、日志 `/logs`、综合统计 `/stats` |
| `sql_config.py` | `/api/sql-config` | SQL 模板管理 API：增删改查 SQL 模板、管理实例绑定关系 |

### 配置（config.py）

- 双环境支持：**云环境**（3 个实例：浙江网超/浙江乐采网超/湖南乐采网超）和 **乐采云环境**（2 个实例：云南乐采云/广西乐采云）
- 实例中文名称映射、规则 ID 映射
- iData Cookie 配置
- 分页拉取参数（每页 500 条）

## 四、核心数据流

```
┌─────────────┐     SQL 查询      ┌──────────────┐
│   iData     │ ◄──────────────── │  Flask 后端   │
│  数据平台    │   (带 Cookie)    │  data_fetch  │
└─────────────┘                   └──────┬───────┘
                                         │
                    ┌────────────────────┼────────────────────┐
                    ▼                    ▼                    ▼
              ┌─────────┐        ┌────────────┐        ┌──────────┐
              │ SQLite  │        │   随机抽样  │        │ 模型B互检 │
              │ raw_data│        │  (可配置%) │        │  (Mock)  │
              └─────────┘        └────────────┘        └──────────┘
                    │
                    ▼
              ┌─────────┐
              │ 标注任务  │ ◄── 标注员在 annotation_list.html 中标注
              │annotation│      （正确/错误 + 错误标签 + 备注）
              └─────────┘
                    │
                    ▼
              ┌─────────┐
              │ 质检修正  │ ◄── 质检员在 qc_center.html 中复核修正
              │  /报表   │
              └─────────┘
```
