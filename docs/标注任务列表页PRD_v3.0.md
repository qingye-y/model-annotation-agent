## 1. 任务列表表格

### 1.1 表格列（10 列）

| # | 列 | 说明 | 宽度 |
|---|-----|------|------|
| 1 | **任务编码** | `MMDD-NNN`，如 `0524-001` | 90px |
| 2 | **审核规则** | 提示词规则名，去除";审核规则"后缀 | 120px |
| 3 | **任务日期** | 任务分配下去的日期 | 90px |
| 4 | **总条数** | 该任务下商品总数 | 60px |
| 5 | **待标注** | 尚未标注的数量（红色高亮） | 60px |
| 6 | ✅ **正确** | 标注为";正确"的条数 | 60px |
| 7 | ❌ **错误** | 标注为";错误"的条数 | 60px |
| 8 | **进度** | 进度条 + 百分比 | 100px |
| 9 | **分配人** | 谁分配的 | 70px |
| 10 | **操作** | ";进入标注"按钮 | 90px |

### 1.2 任务编码规则

```
格式：MMDD-NNN

MMDD  — 任务分配日期（月日），如 0524
NNN   — 当天第几个任务，从 001 开始

示例：
  0524-001  → 5月24日第1个任务
  0524-002  → 5月24日第2个任务
```

> 生成时写入 RawData.task_code 字段，按";任务日期"去重自增。

### 1.3 规则名精简

| 原名称 | 表格显示 |
|--------|---------|
| 浙江网超审核规则 | 浙江网超 |
| 浙江乐采网超审核规则 | 浙江乐采网超 |
| 其他乐采网超审核规则 | 其他乐采网超 |

### 1.4 进度计算

```
进度 = (正确 + 错误) / 总条数 × 100%
```

> 忽略不计入进度，因为标注员基本不用忽略。

## 2. 筛选栏

表格上方筛选栏，4 项：

| 筛选项 | 形式 | 选项/说明 |
|--------|------|----------|
| **搜索框** | 输入框 | 按关键字模糊搜索任务编码或审核规则名 |
| **审核规则** | 下拉 | 全部 / 浙江网超 / 浙江乐采网超 / 其他乐采网超（动态读取 prompt_rules 去";审核规则"后缀） |
| **标注状态** | 下拉 | 全部 / 进行中（待标注>0）/ 已完成（待标注=0） |
| **任务日期范围** | 双日期框 | 起止日期，筛选任务日期在该范围内的任务 |

右侧：查询 / 重置按钮

### 2.1 筛选交互

| 操作 | 行为 |
|------|------|
| 任一筛选条件变更 | 点";查询"生效（不自动搜，避免频繁请求） |
| 点击";重置" | 清空所有条件 → 自动查询 |
| 进入明细 → 返回任务列表 | 保留筛选条件和排序状态 |
| 筛选后无结果 | 表格显示";暂无匹配的任务" |

---

## 3. 点击进入明细

点某行或";进入标注" → 进入该任务下的商品明细。面包屑：`← 返回任务列表 | 规则名 · 日期`。

明细表格（复用现有）：商品ID / 商品名称 / 类目 / 模型A结果 / 模型B结果 / 标注状态 / 操作（标注/查看）。

---

## 4. 验收标准

| # | 验证项 | 预期 |
|---|--------|------|
| AC-1 | 10 列正确 | 无实例列、无忽略列、规则名已精简 |
| AC-2 | 任务编码 `MMDD-NNN` | 0524-001, 0524-002... |
| AC-3 | 规则名精简 | 浙江网超 / 浙江乐采网超 / 其他乐采网超 |
| AC-4 | 进度条 | 百分比 = (正确+错误)/总条数 |
| AC-5 | 搜索框筛选 | 输入";乐采"→ 只显示规则名含";乐采"的任务 |
| AC-6 | 审核规则筛选 | 选";浙江网超"→ 只显示该规则任务 |
| AC-7 | 标注状态筛选 | 选";进行中"→ 只显示待标注 > 0 的任务 |
| AC-8 | 日期范围筛选 | 选 0521~0524 → 只显示任务日期在此范围内的 |
| AC-9 | 重置 | 清空所有条件 → 全量显示 |
| AC-10 | 进入明细 | 点击 → 正常展示该任务下商品 |
| AC-11 | 返回保留筛选 | 从明细返回 → 筛选条件不丢失 |
