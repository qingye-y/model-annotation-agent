## 一、模块定位

任务调度模块负责两件事：
1. **生成标注任务**：管理员在机审中心手动触发，将互检完成的数据按「实例+规则」聚合并写入任务池
2. **分配标注任务**：管理员将任务池中的数据分配给标注员进行人工判定

**位置**：机审任务中心（生成任务）→ 任务调度中心（分配任务）→ 标注列表页（标注员工作台）

> ⚠️ 互检完成后数据**不会自动进入任务池**，需管理员手动触发「生成标注任务」。

---

## 二、核心流程

```
机审中心批次完成互检
    ↓
管理员在机审中心点击「生成标注任务」  ← v1.4 新增：手动触发
    ↓
弹窗配置：
  ├── 选择规则（从提示词规则管理中读取）
  ├── 选择实例（自动带入该规则绑定的实例）
  └── 抽检比例（默认5%，可手动调整）
    ↓
数据按「实例+规则」聚合写入任务池
    ↓
任务调度中心可看到待分配任务
    ↓
管理员分配任务给标注员
    ↓
标注员在「我的任务」页面看到任务
    ↓
标注员完成标注 → 结果落库 → 进入质检/Badcase流程
```

## 三、待标注任务池

### 3.1 数据来源

| 数据来源 | 说明 | 进入任务池比例 |
|----------|------|---------------|
| 模型 A/B 不一致的数据 | 互检结果不同的记录 | 100%，全部进入 |
| 模型 A/B 一致的数据 | 互检结果相同的记录 | 按抽检比例（默认 5%），随机抽取 |

**抽检比例配置**：
- **默认值**：在「系统配置-任务调度配置」中设置全局默认值（如 5%）
- **手动调整**：在机审任务中心「生成标注任务」时，弹窗中自动带入默认值，管理员可手动修改

### 3.2 聚合维度（v1.4 修正）

任务池按 **「实例 + 规则」** 二维聚合。同一条规则在不同实例下各自独立展示。

| 维度 | 来源 | 说明 |
|------|------|------|
| 实例 | `GET /api/config/instance-rule-mapping` | 该规则绑定了哪些实例 |
| 规则 | `GET /api/prompt-rules` | 提示词规则管理中的规则列表 |

> **示例**：「浙江乐采网超审核规则」绑定了 ZJWC 和 HWCS → 任务池显示两条：ZJWC-浙江乐采网超审核规则、HWCS-浙江乐采网超审核规则。

#### 违规规则分类

不一致数据按违规原因 CASE WHEN 分类（与看板违规原因饼图同源），一致性抽检数据统一归为「合规抽检」。

> **两个概念区分**：任务池聚合用的「规则」来自提示词规则管理（如「浙江乐采网超审核规则」），标注员打标时选的「错误标签」来自违规原因 CASE WHEN（如「水印」「站外引流」），两者不同。任务池按前者（审核规则）聚合。

### 3.3 "生成标注任务"按钮放置（v1.4 新增）

两处都需要：

| 位置 | 按钮 | 行为 |
|------|------|------|
| **机审任务中心** 批次列表行（model_task.html） | 「生成标注任务」 | 打开弹窗，选择规则+实例+抽检比例 → 提交 |
| **批次详情页** 操作栏（batch_detail.html） | 「生成标注任务」 | 同上 |

**已生成过的批次**：

| 状态 | 按钮变化 |
|------|---------|
| 未生成 | 显示「生成标注任务」 |
| 已生成 | 显示「查看生成明细」+「重新抽取」 |
| 点击「查看生成明细」 | 跳转到任务调度中心，筛选该批次的任务池数据 |
| 点击「重新抽取」 | 弹窗确认 → 清空旧任务池数据 → 重新按新比例抽取 |

### 3.4 数据来源标记

| 角色 | 能否看到数据来源 | 说明 |
|------|----------------|------|
| 标注员 | ❌ 不显示 | 防止标注员对不一致数据产生预判 |
| 管理员 | ✅ 后台可见 | 每条任务标注「不一致」或「一致性抽检」的来源标签 |

> **安全要求**：标注员不可见数据来源**不能仅靠前端隐藏**——标注员 API 返回数据时，`source_type` 字段不应出现在响应中。管理员 API 正常返回。

### 3.5 任务池展示

任务池返回格式：

```json
{
  "rule_name": "浙江乐采网超审核规则",
  "instance": "ZJWC",
  "total": 50,
  "assigned": 10,
  "remaining": 40,
  "batch_id": "BAT-xxx",
  "data_date": "2026/05/18 ~ 2026/05/20"
}
```

前端表格列：

| 列 | 说明 |
|----|------|
| 实例 | ZJWC / HWCS / ... |
| 规则名称 | 如「浙江乐采网超审核规则」 |
| 原始数据量 | 该规则×实例下待标注总条数 |
| 已分配量 | 已经分配给标注员的数量 |
| 剩余待分配 | 还没有分配的数量 |
| 数据日期 | 取自 FetchLog.data_start_date ~ data_end_date |
| 操作 | 分配按钮 |

## 四、任务分配

### 4.1 分配方式

| 方式 | 说明 |
|------|------|
| 管理员手动分配 | 管理员选择任务，指定标注员和数量（唯一方式） |

### 4.2 分配算法

| 算法 | 说明 |
|------|------|
| 平均分配 | 将任务均匀分给选定的标注员 |
| 按剩余额度比例分配 | 根据每个标注员当天还剩多少额度，按比例分 |
| 清空分配 | 撤回该规则下已分配但未开始的任务 |

### 4.3 分配约束

| 约束 | 说明 |
|------|------|
| 一人一条 | 同一条数据只能分配给一个人，不会被多人重复标注 |
| 不超额度 | 分配给某人的数量不能超过他当天剩余额度 |
| 按实例+规则分配 | 以「实例+规则」为单位进行分配。同一条规则在不同实例下独立分配 |
| 额度共用 | 不一致数据任务和一致性抽检任务共用标注员每日额度，不分开计算 |

### 4.4 分配记录

每次分配生成一条记录。字段：

| 字段 | 说明 |
|------|------|
| 分配时间 | 操作时间戳 |
| 操作人 | 管理员用户名 |
| 实例 | 分配的实例编码（如 ZJWC） |
| 规则 | 分配的审核规则名（来源：提示词规则管理，如「浙江乐采网超审核规则」） |
| 标注员 | 分配给了谁 |
| 数量 | 分配了多少条 |
| 分配方式 | 平均分配 / 按额度比例 |
| 数据类型 | 不一致数据 / 一致性抽检 |
| 关联批次 | 生成任务时关联的 FetchLog batch_id |

> 分配记录在前端「分配历史」面板中展示，管理员的 dispatch_center 页面可查看。

### 4.5 清空分配处理

「清空分配」撤回已分配但**未标注**的任务：

| 操作 | 效果 |
|------|------|
| 清空分配 | 将 `Annotation` 表中 `is_submitted=False` 的记录标记为取消（`status='revoked'`），对应 RawData 回到待分配池 |
| 已标注的记录 | 不受影响，不回撤 |

## 五、标注员管理

### 5.1 基本信息

| 字段 | 说明 |
|------|------|
| 姓名 | 标注员姓名 |
| 绑定规则 | 该标注员擅长/负责哪些审核规则（多选）。规则列表统一来自 `GET /api/prompt-rules`（提示词规则管理），与生成标注任务、任务池聚合使用同一数据源 |
| 每日额度 | 每天最多标注多少条（一致性和不一致任务共用） |
| 状态 | 启用 / 停用 |

### 5.2 负载概览

| 展示 | 说明 |
|------|------|
| 今日已分配 | 今天分给他的总任务数（含不一致+一致性抽检） |
| 今日已完成 | 他今天已经标注完的数量 |
| 剩余额度 | 每日额度 - 今日已分配 |
| 进度条 | 已完成 / 已分配，颜色按比例变化 |

## 六、标注员工作台（「我的任务」页面）

### 6.1 任务列表

标注员登录后看到的任务列表，包含：

| 列 | 说明 |
|----|------|
| 商品信息 | 商品名称、类目、实例 |
| 实例+规则 | 这条数据属于哪个实例的哪个审核规则（来源：提示词规则管理，如「ZJWC-浙江乐采网超审核规则」） |
| 模型 A 结果 | 合规/违规，及拒绝原因 |
| 模型 B 结果 | 合规/违规，及拒绝原因 |
| 标注状态 | 待标注 / 已标注 |
| 操作 | 标注按钮 |

> **不显示**：数据来源（不一致/一致性抽检），防止预判。

### 6.2 标注弹窗

点击「标注」按钮后打开弹窗，分为左右两栏：

| 区域 | 内容 |
|------|------|
| 左侧 | 商品图片垂直滑动（主图 + 详情图 + SKU图），点击放大 |
| 右侧 | 模型A审核结果（合规/违规 + 拒绝原因）、模型B审核结果（合规/违规 + 拒绝原因） |
| 底部 | 三个大按钮：✅ 正确 / ❌ 错误 / ⬜ 忽略 |

**选「错误」时**：强制弹出错误标签选择器，标签列表从系统配置（label_config）动态读取，必须选一个才能提交。

**提交后**：
- 写入 `Annotation` 表：`result` / `error_tag` / `note` / `is_submitted=True`
- 更新 `RawData` 表：`check_result` / `annotation` / `annotator` 字段
- 键盘快捷键：← → 切换任务，ESC 关闭弹窗

### 6.3 筛选

| 筛选项 | 说明 |
|--------|------|
| 规则类型 | 按违规规则筛选 |
| 标注状态 | 待标注 / 已标注 |
| 实例 | 按实例筛选 |
