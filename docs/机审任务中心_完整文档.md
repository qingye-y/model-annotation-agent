## 一、产品概述

### 1.1 产品定位

机审任务中心是商品审核大模型质检标注工作台的核心模块，为标注团队提供从 iData 数据平台自动获取机审数据（AI 审核记录）的能力，并支撑数据可视化看板和 SQL 配置管理功能。

### 1.2 目标用户

| 用户角色 | 使用场景 |
|----------|----------|
| **标注员** | 查看批次数据，进行质检标注 |
| **运营人员** | 拉取数据、查看看板、导出报表 |
| **运维人员** | 配置 SQL 模板、管理 Cookie |
| **管理员** | 全量管理（包含以上所有权限） |

### 1.3 核心价值

- **效率提升**：从人工 SQL 导出 → 全自动定时/手动拉取
- **数据一致性**：分层抽样保证样本代表性
- **实时可见**：看板实时展示违规率、Top 原因、不一致率
- **灵活配置**：SQL 模板在线修改，无需代码发布

## 三、详细功能说明

### 3.1 F1 线上数据拉取

#### F1.1 环境与实例选择

- 两组单选按钮：云环境 / 乐采云环境
- 选择后，下方实例多选框动态更新
- 云环境可选：浙江网超(ZJWC)、浙江乐采网超(HWCS)、湖南乐采网超(HNLCWC)
- 乐采云环境可选：云南乐采云(YNLCY)、广西乐采云(GXLCY)
- 支持全选/取消全选

#### F1.2 日期范围选择

- 两个日期选择器：开始日期、结束日期
- 默认值：开始日期=7天前，结束日期=昨天
- 校验：开始日期 ≤ 结束日期
- 支持快捷选项：近7天、近30天、本月

#### F1.3 抽样比例配置

- 数字输入框 + 滑块，范围 1-100%，默认值 5%
- 抽样算法：**分层抽样** — 合规和违规各自按比例抽取，样本中违规率与线上全量完全一致
- MD5 固定种子保证可复现性，相同参数两次拉取结果完全一致

#### F1.4 执行取数

- 点击后按钮变为「拉取中...」，禁止重复点击
- 显示进度：「正在拉取 ZJWC... (1/3)」+ 进度条
- Cookie 过期 → 弹窗提示；网络超时 → 显示错误信息

#### F1.5 取数结果展示

- 批次号（如 BAT-20260525103456）
- 线上总数 / 合规数 / 违规数 / 本次拉取数
- 操作按钮：「查看详情」「查看看板」

## 二、核心模块详解

### 2.1 取数服务（fetch_service.py）

#### 2.1.1 分层抽样算法

**问题**：简单随机抽样会导致样本违规率与线上全量偏差大。

**方案**：分层抽样，按合规/违规分层，按相同比例从各层抽取。

```
输入：total=10000, compliant=8000, non_compliant=2000, pct=5%
处理：
  合规样本 = 8000 × 5% = 400
  违规样本 = 2000 × 5% = 100
  样本违规率 = 100/500 = 20%（与线上完全一致）

实现：
  - 使用 MD5(审核id) 作为排序种子（固定种子可复现）
  - ROW_NUMBER() OVER (ORDER BY MD5(审核id)) 生成序号
  - Python 端按审核id去重（seen_ids set）+ 分类截断
  - OVERSAMPLE = 1.5x 补偿 LEFT JOIN 行倍增
```

#### 2.1.2 LEFT JOIN 行倍增问题

**问题**：`detail_sql` 包含 `LEFT JOIN dwd_itm_audit_reject_detail_y d`，一个 app_id 对应多条驳回详情时，LEFT JOIN 产生重复行，导致 COUNT 虚高。

**解决方案**：

| 函数 | 修复方式 |
|------|----------|
| `fetch_data_from_idata` | `COUNT(*)` → `COUNT(DISTINCT 审核id)` |
| `fetch_error_reasons_online` COUNT查询 | 同上 |
| `fetch_error_reasons_online` 聚合SQL | 内层 `GROUP BY 审核id, 创建日期, violation_tag` + 外层 `SUM(cnt)` |
| `fetch_data_with_template` | `COUNT(*)` → `COUNT(DISTINCT 审核id)` |

**实测**：YNLCY 20260518 修复前 ratio=2.00（精确2倍），修复后完全对齐 iData 基准。

#### 2.1.3 iData API 调用

```python
def execute_sql_query(sql, instance, env):
    cookie = get_idata_cookie()  # 优先从数据库读取，fallback config.py
    headers = {'Content-Type': 'application/json', 'Cookie': cookie}
    payload = {
        'sql': sql,
        'instance': instance,
        'datasourceType': IDATA_DATASOURCE_TYPE
    }
    resp = requests.post(query_api_url, json=payload,

    return resp.json()['data']['values']
```

| **SqlTemplate** | SQL 模板（可配置CRUD） | 运维手动管理 |
| **FetchPipeline** | 管道执行顺序配置 | 初始化预置 |
| **User** | 用户账户 | 长期保留 |
| **Annotation** | 标注记录 | 长期保留 |
| **QcRecord** | 质检记录 | 长期保留 |

### 3.2 RawData 字段映射

