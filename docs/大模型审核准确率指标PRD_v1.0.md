## 1. 背景与目标

### 1.1 背景

数据看板的"大模型审核准确率"模块至今无数据。该指标是衡量模型A审核质量的核心指标，需定义准确率的计算逻辑，打通从数据拉取、互检、人工标注到看板展示的全链路。

### 1.2 目标

- 定义准确率的无歧义计算规则
- 在看板准确率卡片中展示数值（替代"暂无数据"）
- 支持准确率趋势图（近7天/近30天）
- 支持按实例筛选联动
- 人工标注结果变更后，刷新看板数值随之更新

---

## 2. 准确率计算逻辑

### 2.1 核心公式

```
大模型审核准确率 = 模型A审核正确的记录数 / 参与互检的抽样记录总数
```

### 2.2 参与范围

- 仅纳入已完成互检（`modelb_reviewed = True`）且模型B有结果的记录
- 按商品的业务日期（`gmt_created_time` 截取日期）和实例（`instance_code`）进行分组统计
- 漏审数据（`modelb_reviewed = False` 或 `modelb_consistent = null`）：暂不计入，直到被人工标注后再纳入

### 2.3 模型A正确判定规则

对于每一条参与互检的记录，模型A是否正确按以下规则判定：

#### 规则一：互检一致（模型A = 模型B）

- **默认**：模型A正确
- **修正**：若该记录已被**人工标注**且标注结果为"错误"，则修正为**模型A错误**

#### 规则二：互检不一致（模型A ≠ 模型B）

- **默认**：模型A错误
- **修正**：若该记录已被**人工标注**且标注结果为"正确"，则修正为**模型A正确**

#### 规则三：漏审数据

- **暂不计入**准确率分母，直到被人工标注
- 标注**正确** → 计入分母（+1），计入分子（+1）
- 标注**错误** → 计入分母（+1），不计入分子

### 2.4 判定矩阵

| 互检状态 | 人工标注结果 | 模型A正确？ | 计入分母 | 计入分子 |
|---------|------------|------------|---------|---------|
| 一致（modelb_consistent=True） | 无 | ✅ 正确 | ✅ | ✅ |
| 一致 | correct | ✅ 正确 | ✅ | ✅ |
| 一致 | error | ❌ 错误 | ✅ | ❌ |
| 一致 | ignore | ✅ 正确 | ❌（忽略） | ❌ |
| 不一致（modelb_consistent=False） | 无 | ❌ 错误 | ✅ | ❌ |
| 不一致 | correct | ✅ 正确 | ✅ | ✅ |
| 不一致 | error | ❌ 错误 | ✅ | ❌ |
| 不一致 | ignore | ❌ 错误 | ❌（忽略） | ❌ |
| 漏审（modelb_consistent=null） | 无 | — | ❌ | ❌ |
| 漏审 | correct | ✅ 正确 | ✅ | ✅ |
| 漏审 | error | ❌ 错误 | ✅ | ❌ |

## 3. 展示方式

### 3.1 看板指标卡片

- 位置：数据看板首页，准确率指标卡片
- 展示内容：当前日期范围内所有抽样数据的整体准确率百分比
- 无数据时：显示"暂无互检数据"（灰字）
- 有数据时：显示百分比数字 + 辅助文字"（正确数 / 总互检数）"
- 格式：`87.5%（1,234 / 1,410）`

### 3.2 准确率趋势图

- 位置：数据看板，与违规率趋势图并列展示
- 时间范围：支持近7天 / 近30天切换（与违规率趋势图共用切换按钮）
- X轴：日期（YYYY-MM-DD）
- Y轴：准确率百分比（0%~100%）
- 筛选联动：按实例筛选时，趋势图同步更新为选中实例的数据
- 无数据日期：趋势图显示为空或断点，不补零

### 3.3 趋势图详细数据（hover）

hover tooltip 显示：

```
2026-05-25
准确率：87.5%
正确：1,234 条 / 总互检：1,410 条
```

## 4. 数据来源

| 数据项 | 来源字段 | 说明 |
|--------|---------|------|
| 互检一致性 | `RawData.modelb_consistent` | true=一致，false=不一致，null=漏审 |
| 人工标注结果 | `RawData.check_result` | correct / error / ignore / null |
| 模型B有无结果 | `RawData.modelb_reviewed` | true=已互检，false=未互检 |
| 商品业务日期 | `RawData.gmt_created_time` | 截取日期作为分组维度 |
| 实例 | `RawData.instance_code` | 筛选维度 |

> 注：`check_result` 字段存储的是标注员对"模型A判定是否正确"的直接判定结果。
> - `correct` = 模型A判定正确
> - `error` = 模型A判定错误
> - `ignore` = 忽略（不纳入准确率计算）

## 5. 验收标准

| # | 验收条件 | 验证方式 |
|---|---------|---------|
| V1 | 无互检数据时，卡片显示"暂无互检数据" | 清除本地数据后刷新看板 |
| V2 | 完成互检后（无人工标注），卡片按默认规则显示准确率 | DB 中插入 `modelb_consistent=True` 记录，刷新看板验证 |
| V3 | 人工标注某条记录后，刷新看板，准确率随之变化 | 在标注列表中标注一条记录，刷新看板对比前后数值 |
| V4 | 标注结果从正确→错误，准确率应下降 | 标注→刷新→重新标注→刷新，对比两次准确率 |
| V5 | 标注结果从错误→正确，准确率应上升 | 同上反向验证 |
| V6 | 趋势图可按天展示准确率 | 切换到有多天数据的日期范围，观察趋势图 |
| V7 | 趋势图支持近7天/近30天切换 | 点击切换按钮，观察 X轴范围变化 |
| V8 | 按实例筛选时，卡片和趋势图只反映选中实例 | 在实例筛选器中选中单个实例，验证两个图表的数值一致性 |
| V9 | 忽略（ignore）记录不计入准确率分母 | 标注一条为 ignore，验证卡片分母不变 |
