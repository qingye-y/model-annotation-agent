## 一、项目概述

### 1.1 这是什么系统

本工作台是一个**面向商品审核大模型质检的全流程标注工具**，用于：

1. 从 iData 数据平台自动拉取 AI 审核数据（商品审核记录）
2. 标注员对 AI 审核结果进行人工质检标注（判断 AI 对不对）
3. Model B 双模型互检，标记不一致样本
4. 质检员复核修正错误标注
5. Badcase 管理：错误样本归因分析，持续优化模型

### 1.2 整体架构

```
┌─────────────────────────────────────────────────────┐
│  前端（纯静态 HTML + Jinja2 模板）                     │
│  templates/ + static/                                │
│  → 15 个页面，通过 index.html 主框架的 iframe 加载     │
└────────────────────┬────────────────────────────────┘
                     │ HTTP REST API
┌────────────────────▼────────────────────────────────┐
│  Flask 后端（app.py + 8 个蓝图）                       │
│  blueprints/                                        │
│  → 用户认证 / 数据拉取 / 看板统计 / SQL配置 / 互检等    │
└────────────────────┬────────────────────────────────┘
                     │ SQLAlchemy ORM
┌────────────────────▼────────────────────────────────┐
│  SQLite 数据库（app.db）                              │
│  → FetchLog / RawData / DailyStats / Annotation 等   │
└────────────────────┬────────────────────────────────┘
                     │ HTTP JSON（带 Cookie）
┌────────────────────▼────────────────────────────────┐
│  iData 数据平台（idata.cai-inc.com）                   │
│  → Hive 数仓：AI审核结果表 / 商品基础表 / 驳回详情表   │
└─────────────────────────────────────────────────────┘
```

## 三、部署指南

### 3.1 环境要求

- Python 3.8 ~ 3.13
- 网络可达 `idata.cai-inc.com`（iData 数据平台）
- 无需安装 MySQL/Redis 等额外服务（使用 SQLite）

### 3.2 快速安装（5步完成）

**Step 1：复制项目到本地**

将整个 `模型标注agent` 文件夹复制到目标机器。

**Step 2：安装 Python 依赖**

```bash
cd 模型标注agent
pip install -r requirements.txt
```

**Step 3：配置 Cookie（必须）**

打开 `config.py`，找到第 16 行 `IDATA_COOKIE`，替换为你的账号 Cookie：

```python
# config.py 第 16 行
IDATA_COOKIE = "your-new-cookie-here"
```

获取 Cookie 方法：
1. 用浏览器登录 iData：https://idata.cai-inc.com
2. 按 F12 打开开发者工具
3. 切换到 **Network（网络）** 标签
4. 刷新页面，点击任意一个网络请求
5. 在右侧 **Headers** → **Request Headers** 中找到 `Cookie:`
6. 复制完整值（很长），粘贴到 config.py

**Step 4：修改密钥（建议）**

```python
# config.py 第 4 行
SECRET_KEY = 'your-own-random-secret-key'
```

**Step 5：启动**

```bash
python3 app.py
```

看到以下输出即成功：
```
* Running on http://127.0.0.1:5000/
* Running on http://localhost:5000/
```

**Step 6：打开浏览器访问**

```
http://localhost:5000
```

默认管理员账号：
| 账号 | 密码 | 角色 |
|------|------|------|
| admin | admin123 | 管理员 |
| liziyue | admin123 | 标注员 |

> ⚠️ 首次登录后请立即在「账号管理」中修改 admin 密码！

### 3.3 部署到服务器（可选）

如果要将服务暴露给局域网或公网：

```python
# app.py 最后一行改为：
app.run(host='0.0.0.0', port=5000, debug=False)
```

然后：
```bash
# 使用 systemd 管理进程（推荐）
# 或使用 nohup / screen / pm2 后台运行
nohup python3 app.py > flask.log 2>&1 &
```

> ⚠️ 安全提醒：公网暴露时请务必修改 SECRET_KEY 和 admin 密码！

## 四、功能操作手册

### 4.1 机审任务中心（数据拉取）

**路径**：左侧菜单 → 机审任务中心

**功能**：从 iData 数据平台自动拉取 AI 审核数据。

**操作步骤**：

1. 选择**环境**（云环境 / 乐采云环境）
   - 云环境：浙江网超(ZJWC)、浙江乐采网超(HWCS)、湖南乐采网超(HNLCWC)
   - 乐采云环境：云南乐采云(YNLCY)、广西乐采云(GXLCY)

2. 勾选要拉取的**实例**（可多选）

3. 选择**日期范围**（开始日期 ~ 结束日期）

4. 设置**抽样比例**（默认 5%）
   - 含义：从线上全量数据中按比例抽取样本
   - 分层抽样：合规和违规各自按比例抽取，样本中违规率与线上完全一致

5. 点击「**立即拉取**」

6. 查看**结果摘要**：
   - 批次号：唯一标识（如 BAT-20260527123456）
   - 线上总数：该日期范围在 iData 的全量审核数
   - 本次拉取：实际存入本地数据库的条数

7. 点击「**查看详情**」→ 跳转到批次详情页

**常见问题**：

| 现象 | 原因 | 解决方法 |
|------|------|----------|
| 提示「Cookie 已过期」| iData Cookie 失效 | 重新获取 Cookie 并更新 config.py |
| 提示 401/403 | 无权限访问该实例 | 确认账号有对应实例权限 |
| 数据量为 0 | 日期范围内无数据 | 扩大日期范围 |
| 部分实例失败 | 单个实例查询超时 | 单独重试失败的实例 |

### 4.2 批次详情

**路径**：机审任务中心 → 批次列表 → 「查看详情」按钮

**功能**：查看单个批次的完整数据明细。

**概览卡片**：
- 线上总数 / 本次拉取 / 合规数 / 违规数
- 数据日期范围 / 创建时间 / 互检状态

**明细筛选**：
- AI 审核结果：全部 / 合规 / 违规
- 实例：多选
- 关键词搜索：商品名称 / 违规原因
- 互检状态：全部 / 已互检 / 未互检 / 一致 / 不一致

**导出功能**（操作栏下拉菜单）：
- **原始全量**：该批次全部数据，28 个字段
- **违规数据**：仅 ai_result='违规' 的记录
- **互检差异**：仅已互检且不一致的记录，含 Model B 结果

### 4.3 数据看板

**路径**：左侧菜单 → 数据看板

**功能**：实时查看违规率、不一致率、Top 违规原因、每日趋势。

**KPI 卡片**：
- 总审核数（蓝）| 合规数（绿）| 违规数（红）| 违规率 | 不一致率（橙）

**违规原因饼图**：
- Top 10 按频次降序展示
- 超过 10 种原因时，第 11 种及以后归入「**其他**」
- 点击「其他」可展开查看被合并的所有原因

**每日趋势图**：
- X 轴：日期
- 折线：总审核数 / 违规数 / 违规率

**筛选**：
- 日期范围（默认近30天）
- 实例（可多选）

### 4.4 标注任务

**路径**：左侧菜单 → 标注任务

**前置条件**：管理员在「任务调度中心」已生成标注任务。

**操作步骤**：

1. 左侧筛选栏选择**规则名称**、**标注状态**、**实例**
2. 表格中点击任意商品行的「**标注**」按钮
3. 弹出详情弹窗：
   - 左侧：商品图片（主图/详情图/SKU图，点击放大）
   - 右侧：AI审核结果、AI拒绝原因、商品链接（可点击跳转查看原商品）
4. 判断 AI 审核是否正确：
   - ✅ **正确**：AI 判断准确，无需修正
   - ❌ **错误**：AI 判断有误，需要选择错误标签 + 填写备注
5. 点击「**提交标注**」

**错误标签选项**：
- 水印 | 类目错放 | 盗图 | 价格违规 | 虚假宣传 | 品牌侵权 | 违禁词 | 其他

### 4.5 任务调度中心

**路径**：左侧菜单 → 任务调度中心

**功能**：管理标注规则、生成标注任务、分配给标注员。

**Tab 说明**：
- **规则调度**：创建/编辑/启用规则，生成标注任务
- **实例调度**：按实例维度生成任务
- **任务池**：查看当前待标注任务数量
- **我的任务**：当前登录标注员的任务概览
- **任务组**：任务分组管理
- **分配历史**：任务分配记录

**生成任务流程**：
1. 选择规则（对应审核标准）
2. 选择实例
3. 选择日期范围
4. 设置抽检比例
5. 选择标注员
6. 点击「生成任务」

### 4.6 质检中心

**路径**：左侧菜单 → 质检中心

**功能**：质检员对标注员的标注结果进行复核。

**操作**：与标注任务类似，区别在于判断的是「人审标注是否正确」而非「AI审核是否正确」。

### 4.7 SQL 配置管理

**路径**：左侧菜单 → SQL 配置（系统配置子菜单）

**功能**：在线修改取数 SQL 模板，无需改代码。

**⚠️ 谨慎操作**：修改 SQL 可能导致取数结果不正确或取数失败。

**参数说明**：
| 参数 | 说明 | 示例 |
|------|------|------|
| `${start_date}` | 数据开始日期 | 20260518 |
| `${end_date}` | 数据结束日期 | 20260525 |
| `${instance}` | 实例编码 | ZJWC |
| `${year}` | 年份 | 2026 |
| `${detail_sql}` | 基础明细SQL（系统注入）| — |

### 4.8 账号管理

**路径**：左侧菜单 → 账号管理（系统配置子菜单）

**功能**：增删改查用户账号、设置日配额、重置密码。

**默认账号**：
- admin（管理员）：所有权限
- liziyue（标注员）：标注权限

## 六、配置文件说明（config.py）

### 6.1 必改配置（部署到新环境必须修改）

| 配置项 | 行号 | 说明 | 修改建议 |
|--------|------|------|----------|
| `SECRET_KEY` | 4 | Flask 会话密钥 | 改为随机字符串 |
| `IDATA_COOKIE` | 16 | iData 认证 Cookie | 必须替换为有效值 |

### 6.2 可选配置

| 配置项 | 行号 | 说明 | 默认值 |
|--------|------|------|--------|
| `FETCH_PAGE_SIZE` | 9 | 每次 iData 请求条数上限 | 500 |
| `FETCH_CONCURRENCY` | 10 | 并发拉取线程数 | 3 |
| `IDATA_DATASOURCE_TYPE` | 34 | iData 数据源类型 | hive |
| `ENV_CONFIG` | 22-31 | 环境与实例映射 | 云环境+乐采云 |
| `INSTANCE_NAMES` | 37-43 | 实例中文名映射 | 5个实例 |

### 6.3 环境变量方式配置（推荐生产环境使用）

避免在代码中明文存储敏感信息：

```bash
# Linux/macOS
export SECRET_KEY='your-production-secret-key'
export IDATA_COOKIE='your-cookie-here'
export DATABASE_URL='sqlite:///app.db'
python3 app.py
```

## 八、常见问题排查

### 8.1 启动相关

**Q：报错 `ModuleNotFoundError: No module named 'flask'`**

```bash
pip install -r requirements.txt
```

**Q：报错 `sqlite3.OperationalError: database is locked`**

原因：多个进程同时写入数据库。
解决：确保只有一个 `python3 app.py` 进程在运行。

**Q：端口 5000 被占用**

修改 `app.py` 最后一行：
```python
app.run(host='0.0.0.0', port=5001)  # 改为其他端口
```

### 8.2 取数相关

**Q：拉取提示 `401 Unauthorized`**

Cookie 已过期，重新获取并更新 `config.py`。

**Q：拉取提示 `Connection timeout`**

iData 查询超时（默认 5 分钟），可能是数据量过大或网络问题。
解决：缩小日期范围后重试。

**Q：合规/违规数为 0**

检查 SQL 中 AI审核结果 字段的映射，确认 iData 返回数据格式未变。

**Q：样本违规率与线上全量偏差很大**

这是严重问题！正常情况下分层抽样比例应完全一致。
原因可能是 iData 返回数据格式变化（AI审核结果字段值不是 '合规'/'违规'），
或 LEFT JOIN 产生了行倍增未正确去重。

### 8.3 标注相关

**Q：标注员登录后看不到任务**

检查：管理员是否在「任务调度中心」已生成并分配任务。

**Q：提交标注提示「请选择正确/错误」**

标注操作必须选择审核结果（正确/错误）才能提交。
