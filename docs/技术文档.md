## 三、iData 取数管道（S1-S6）

> **修改约定**：云环境（ZJWC/HWCS/HNLCWC）和乐采云环境（YNLCY/GXLCY）各有一套 SQL 模板，**修改时必须同步更新两个模板**。

### 3.1 步骤说明

| 步骤 | 名称 | SQL模板category | 说明 |
|------|------|----------------|------|
| S1 | 取数基础SQL | —（硬编码） | 拼接 WHERE 条件，返回 RawData 基础字段 |
| S2 | COUNT 总数统计 | `count` | COUNT(DISTINCT 审核id) 统计合规/违规总数 |
| S3 | 合规数据抽样 | `sample` | 翻页抽取合规数据（按 position 过滤） |
| S4 | 违规数据抽样 | `sample` | 翻页抽取违规数据（按 position 过滤） |
| S5 | 违规原因聚合 | `reason` | 内层 GROUP BY + 外层 SUM，防止 LEFT JOIN 行倍增 |
| S6 | 每日分组统计 | `daily` | 按创建日期 GROUP BY COUNT，用于 DailyStats 按日覆写 |

### 3.2 关键函数

| 函数 | 文件 | 说明 |
|------|------|------|
| `get_pipeline_sql_by_category(env, category)` | `services/fetch_service.py` | 按 category 读取第一条已启用管道模板 |
| `replace_pipeline_params(sql, **kwargs)` | `services/fetch_service.py` | 统一替换 `{detail_sql}` / `{positions}` 占位符 |
| `generate_daily_stats(..., daily_counts_by_date=None, error_reasons_by_date=None)` | `services/fetch_service.py` | 生成 DailyStats，优先使用线上全量数据 |

### 3.3 COUNT LEFT JOIN 行倍增问题（已修复）

**问题**：所有 COUNT 聚合查询中，`detail_sql` 含 `LEFT JOIN dwd_itm_audit_reject_detail_y`，同一 app_id 有多条驳回详情时 JOIN 产生重复行，统计虚高。

**修复位置**：

| 函数 | 行 | 修复方式 |
|------|----|---------|
| `fetch_data_from_idata` | ~701 | `COUNT(*)` → `COUNT(DISTINCT 审核id)` |
| `fetch_error_reasons_online` COUNT查询 | ~711-712 | 同上 |
| `fetch_error_reasons_online` 违规原因聚合 | ~979-991 | 内层 `GROUP BY 审核id,创建日期,violation_tag COUNT(*)` + 外层 `SUM(cnt)` |
| `fetch_error_reasons_online` 日期条件 | ~904 | `if start_date` → `if end_date`（条件写反bug） |
| `fetch_data_with_template` | ~977-984 | `COUNT(*)` → `COUNT(DISTINCT 审核id)` |

### 4.2 编码处理

reject_reason 字段内部是 **latin1 编码的 UTF-8 字节序列**，需双重解码：

```
Layer 1: raw.decode('utf-8')
Layer 2: fix_encoding(raw_1, latin1→utf-8)
```

### 4.3 fetch_error_reasons_online 返回格式

```python
{
    'by_date': {
        '2026-05-18': {'reason_A': 120, 'reason_B': 45},
        ...
    },
    'global': {'reason_A': 800, 'reason_B': 300}
}
```

- `by_date`：按 `创建日期` 字段分区，用于写入 DailyStats
- `global`：全量聚合，用于日志和调试

### 5.1 common.js — 全局共享 API 工具

**路径**：`static/common.js`

| 函数 | 说明 |
|------|------|
| `safeFetch(url, options)` | 安全 fetch：自动识别 HTML 登录页响应并跳转；处理 401/403；非 JSON 响应容错 |
| `apiGet(url)` | GET 包装 |
| `apiPost(url, data)` | POST 包装（JSON body） |
| `apiPut(url, data)` | PUT 包装（JSON body） |
| `apiDelete(url)` | DELETE 包装 |

**行为说明**：
- 响应 `Content-Type` 非 JSON 且 HTTP 状态非 200 → 判定为 HTML 登录页，跳转 `/login`
- 401 → 跳转登录（带 `next` 参数）
- 403 → toast 提示权限不足
- 非 200 → 解析错误信息返回
- JSON 解析失败 → 跳转登录
