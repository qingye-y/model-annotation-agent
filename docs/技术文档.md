## 四、核心数据流

### 4.1 数据拉取流程

```
POST /api/data-fetch
  │
  ├─ 1. 清理该实例该日期范围旧 RawData（允许重新拉取覆盖）
  ├─ 2. for each instance:
  │     ├─ build_sql() → 动态构建 iData 查询 SQL
  │     ├─ COUNT(DISTINCT 审核id) 获取合规/违规总数
  │     ├─ 分层抽样: 合规 max(1, int(N×pct/100)) + 违规同理
  │     ├─ 超额拉取(1.5x) + 分批查询(300行/批) × 2(合规+违规)
  │     ├─ 按 审核id 去重 + 按目标数截断
  │     └─ 写入 RawData (关联 fetch_batch_id)
  ├─ 3. fetch_error_reasons_online() → 违规原因分布
  ├─ 4. generate_daily_stats() → 写入 DailyStats
  └─ 5. 写入 FetchLog → 返回批次 summary
```

### 4.2 SQL 关键修复（v1.0.0）

| 问题 | 根因 | 修复 |
|------|------|------|
| LEFT JOIN 变 INNER JOIN | `d.pt = '2026'` 在 WHERE 子句 | 移到 `ON r.app_id = d.app_id AND d.pt = '2026'` |
| COUNT 膨胀 | LEFT JOIN 重复行 | `COUNT(DISTINCT 审核id)` |
| 抽样偏差 47% | iData 单次返回 ≤500 行 | 分批 300 行/批 + 1.5x 超额 + 去重截断 |
| 抽样返 0 | existing_audit_ids 全量排除 | 改为拉取前清理旧 RawData |

### 4.3 删除联动

```
DELETE /api/task-batches/<id>
  ├─ Annotation (关联该批次 RawData)
  ├─ DailyStats (按 batch_id)
  ├─ RawData (按 fetch_batch_id)
  └─ FetchLog (按 batch_id)

DELETE /api/task-batches/clear
  ├─ Annotation → DailyStats → RawData → FetchLog
  └─ 返回各表删除计数
```

### 编码注意

reject_reason 字段内部是 latin1 编码的 UTF-8 字节序列，需双重解码：`raw.decode('utf-8')` + `fix_encoding(latin1→utf-8)`
