## 四、核心数据流

### 4.1 取数管道（S1-S6）

```
取数触发 → for each instance:
  S1: 明细取数 SQL → 获取 app_id 列表
  S2: COUNT 统计 → 合规/违规总数
  S3: 合规分层抽样 → 按目标数抽取合规记录
  S4: 违规分层抽样 → 按目标数抽取违规记录
  S5: 违规原因分布聚合 → 写入 DailyStats.violation_reasons
  S6: 每日分组 COUNT → 写入 DailyStats（by_date 精确计数）
```

**配置化实现**（v1.1.0）：
- `SqlTemplate` 表按 `category` 字段区分：detail / count / sample / reason / daily
- `FetchPipeline` 表按 `env` + `sort_order` 定义执行顺序
- `app.py` 启动时自动创建默认管道配置（幂等）
- 无配置时自动回退硬编码 SQL（向后兼容）

### 4.2 SQL 关键修复

| 问题 | 根因 | 修复 |
|------|------|------|
| LEFT JOIN 变 INNER JOIN | `d.pt = '2026'` 在 WHERE 子句 | 移到 `ON r.app_id = d.app_id AND d.pt = '2026'` |
| COUNT 膨胀 | LEFT JOIN 重复行 | `COUNT(DISTINCT 审核id)` |
| 抽样偏差 47% | iData 单次返回 ≤500 行 | 分批 300 行/批 + 1.5x 超额 + 去重截断 |
| 抽样返 0 | existing_audit_ids 全量排除 | 改为拉取前清理旧 RawData |
| 违规原因统计膨胀 | 聚合 SQL 未按审核id去重 | 内层 GROUP BY 审核id+violation_tag → 外层 SUM(cnt) |

### 4.3 模型B互检（v1.1.0 新增并发+带图）

**并发互检**：通过 `concurrency` 配置项（1/2/3/5/8/10/15/20），使用 `ThreadPoolExecutor` 并发调用模型B API。
- 串行（concurrency=1）：主线程逐条执行，保留原逻辑
- 并发（concurrency>1）：主线程预取 dict 列表 → 关闭 session → 线程池并发执行 → 每线程独立 `app.app_context()` + `db.session`

**带图互检**：主图+详情图+SKU图全部传给模型B，实现图文混合审核。
- `build_prompt()` 追加图片数量统计文本
- `call_modelb()` 构建图文混合 `content`（text + image_url 数组）
- JSON 解析：先 `re.sub(r'<think>.*?</think>', '', ..., flags=re.DOTALL)` 剥离推理块，再提取 JSON

### 4.4 删除联动

```
DELETE /api/task-batches/<id>
  ├─ Annotation (关联该批次 RawData)
  ├─ DailyStats (按 batch_id)
  ├─ RawData (按 fetch_batch_id)
  └─ FetchLog (按 batch_id)

DELETE /api/task-batches/clear
  ├─ Annotation → DailyStats → RawData → FetchLog
  └─ 返回各表删除计数
```

## 六、API 接口变更（v1.1.0 新增/变更）

### 6.1 批次详情筛选（变更）

`GET /api/task-batches/<batch_id>/items` 新增 3 个 query param：

| 参数 | 取值 | 说明 |
|------|------|------|
| `review_status` | `reviewed` / `not_reviewed` | 互检状态 |
| `diff_status` | `consistent` / `a_pass_b_reject` / `a_reject_b_pass` | AB差异 |
| `modelb_result` | `合规` / `违规` | 模型B审核结果 |

### 6.2 取数管道步骤（新增）

| 方法 | 路由 | 说明 |
|------|------|------|
| GET | `/api/sql-config?category=count` | 按 category 过滤 SQL 模板 |
| POST | `/api/fetch-pipeline` | 新增管道步骤，自动 sort_order=max+1 |
