## 1. 模块定位

标注员工作台是标注员的**唯一操作入口**，提供标注任务处理、Badcase查看、智能分析报告查阅三大功能。

**定位**：管理员主框架的权限受限版。菜单项由权限配置控制（管理员在 account_management 页面设置）。

**关联模块**：标注员能看到的任务来自「任务调度中心」的分配结果。详见 `docs/任务调度模块PRD_v1.2.md` §六。

## 2. 权限模型

### 2.1 权限项定义

| 权限编码 | 权限名称 | 说明 | 默认值 |
|---------|---------|------|--------|
| `annotation` | 标注任务 | 查看并处理分配给自己的标注任务 | ✅ |
| `badcase_view` | Badcase查看 | 查看被确认为Badcase的记录（只读） | ✅ |
| `analysis_view` | 智能分析查看 | 查看智能分析报告（只读） | ✅ |

> 标注员对 Badcase **只能查看，不能编辑**。编辑权限仅管理员拥有。

### 2.2 默认权限

新标注员创建后，默认拥有全部三项权限。管理员可在 account_management 页面按需关闭某项。

### 2.3 权限存储建议

当前 User 表无权限字段，建议在 `SqlConfig` 表中以 key-value 存储：

```
USER_PERM_{user_id} = {"annotation": true, "badcase_view": true, "analysis_view": true}
```

或给 User 表新增 `permissions` JSON 字段。

## 3. 标注员主框架页面

### 3.1 实现方式

**不新建 `annotator_index.html`**。当前 `index.html` 已支持角色判断：侧边栏菜单项根据用户角色显示/隐藏。标注员登录后跳转到 `/index.html`，侧边栏只显示标注员有权看到的菜单项：

```
┌─────────────────────┐
│  标注员工作台        │
├─────────────────────┤
│  📋 标注任务         │  默认激活，始终显示
│  ⚠️ Badcase管理      │  有 badcase_view 权限时显示
│  📊 智能分析中心      │  有 analysis_view 权限时显示
├─────────────────────┤
│  [标注员: 张三]      │
│  [退出登录]          │
└─────────────────────┘
```

> 当前 `index.html` 已按 `data-parent` 分组，只需在渲染时注入角色信息，JS 端过滤非标注员菜单即可。

### 3.2 登录后角色跳转

| 角色 | 登录后跳转 |
|------|-----------|
| `admin` | `/index.html`（显示全部菜单） |
| `annotator` | `/index.html`（仅显示标注员菜单，默认激活「标注任务」） |

修改 `auth.py` 登录逻辑：

```python
if user.role == 'admin':
    return redirect(url_for('page_index'))
else:
    return redirect(url_for('page_annotation'))  # 直接跳标注任务页
```

## 4. 子页面功能

### 4.1 标注任务页面

**文件**：`templates/annotation_list.html`  
**当前状态**：⚠️ 前端 UI 已完整（图片滑动 + A/B结果对比 + 标注按钮），但使用 **localStorage 模拟数据**，待接入后端 API。

| 功能 | 说明 |
|------|------|
| 查看任务列表 | 只显示分配给当前标注员的任务（API：`GET /api/annotation/my-tasks?annotator_id=xxx`） |
| 标注操作 | 正确 / 错误 / 忽略（API：`PUT /api/annotation/submit`） |
| 筛选任务 | 按规则类型、标注状态（待标注/已标注）、实例 |
| 任务来源标记 | **不显示**（API 不返回 `source_type` 字段） |
| 每日额度+进度 | 显示"今日剩余：X/Y条"（API：`GET /api/dispatch/annotator-load`） |
| 管理员修正通知 | 若曾被修正，标注列表行标记"已被修正"，可点击查看修正前后对比 |

#### 4.1.1 标注弹窗行为（参照现有 UI）

| 区域 | 内容 |
|------|------|
| 左侧 | 商品图片垂直滑动（主图+详情图+SKU图），点击放大 |
| 右侧 | 模型A结果（合规/违规+拒绝原因）、模型B结果（合规/违规+拒绝原因） |
| 底部 | 三个按钮：✅ 正确 / ❌ 错误 / ⬜ 忽略 |
| 选错误时 | 强制弹出标签选择器，标签从 `label_config` 动态读取 |
| 快捷键 | ← → 切换任务，ESC 关闭弹窗 |
| 提交后 | 写入 Annotation 表 + 更新 RawData.check_result，前端列表自动刷新 |

## 5. 数据隔离

| 数据 | 标注员可见范围 | 实现方式 |
|------|-------------|---------|
| 标注任务列表 | 只看到分配给自己的任务 | API 用 `annotator_id = current_user.id` 过滤 |
| Badcase列表 | 所有已确认的Badcase（只读） | API 不限制范围，但前端不提供编辑入口 |
| 任务来源标记 | 不显示 | API 不返回 `source_type` 字段 |
| 其他标注员的标注结果 | 不可见 | API 只返回自己的 Annotation 记录 |

## 7. 标注员可见的 API 列表（v1.3 新增）

| Method | Path | 说明 |
|--------|------|------|
| GET | `/api/annotation/my-tasks` | 获取当前标注员的任务列表（支持筛选/分页） |
| PUT | `/api/annotation/submit` | 提交标注结果 |
| GET | `/api/dispatch/annotator-load` | 获取当前标注员负载（今日已分配/已完成/剩余额度） |
| GET | `/api/badcase/list` | Badcase 列表（只读，按标签/日期筛选） |
| GET | `/api/analysis/annotator-issues` | 智能分析-标注员问题分析 |
| GET | `/api/analysis/search` | 知识库搜索 |
| GET | `/api/qc/notifications` | 管理员修正通知（标注员被通知的记录） |
