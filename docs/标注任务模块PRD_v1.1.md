## 1. 模块定位

标注员工作的核心模块。展示分配给自己的标注任务，点击进入弹窗进行标注操作。

**关联模块**：任务从 dispatch 分配来（annotator_id 写入 Annotation 表），数据流向 → 质检/Badcase。

## 5. 标注弹窗（保留现有 UI，增强）

### 5.1 页面进入

点击任务列表"标注"按钮 → 打开右侧抽屉/弹窗。不跳转页面，保持主列表在后方。

### 5.2 弹窗布局

```
┌──────────────────────────────────────────────┐
│  上一条(←)          标注明细          下一条(→) │
├──────────────────────────────────────────────┤
│  左侧：商品图片区          │  右侧：信息与操作区    │
│  ┌────────────────────┐  │  基本信息            │
│  │ 主图  [放大]        │  │  商品名称 / 类目 / 实例 │
│  │ 详情图 [放大]       │  │  店铺名称            │
│  │ SKU图  [放大]       │  │                     │
│  │ （支持点击放大查看）  │  │  模型A审核结果+拒绝原因 │
│  └────────────────────┘  │  模型B审核结果+拒绝原因 │
│                          │                     │
│                          │  ┌──────────────┐   │
│                          │  │ 标注操作区     │   │
│                          │  │ [✅正确] [❌错误]│   │
│                          │  │ [⬜忽略]       │   │
│                          │  │              │   │
│                          │  │ 错误原因下拉    │   │
│                          │  │ (选错误时展示)  │   │
│                          │  │              │   │
│                          │  │ 备注（手动填写） │   │
│                          │  │              │   │
│                          │  │ [提交并下一条]  │   │
│                          │  └──────────────┘   │
└──────────────────────────────────────────────┘
```

### 5.3 图片展示

| 功能 | 说明 |
|------|------|
| 主图/详情图/SKU图 | 垂直滑动展示（已有） |
| 点击放大 | v1.1 新增：点击图片全屏预览 |
| 无图时 | 显示"暂无图片"占位 |

### 5.4 模型A/B结果展示

| 区域 | 内容 | 方式 |
|------|------|------|
| 模型A结果 | 审核结果（合规/违规）+ 拒绝原因 | 折叠展开 |
| 模型B结果 | 审核结果（合规/违规）+ 拒绝原因 | 折叠展开 |

### 5.5 标注操作

| 操作 | 行为 |
|------|------|
| ✅ 正确 | 自动提交（result=correct），提示成功，**光标跳到下一条** |
| ❌ 错误 | 展开错误原因下拉（必填）+ 备注输入框（选填）→ 点"提交并下一条" |
| ⬜ 忽略 | 自动提交（result=ignore），提示成功，**光标跳到下一条** |

> 错误原因下拉选项从 `label_config` 读取。选错误时未选标签不可提交。

### 5.6 翻页

| 操作 | 行为 |
|------|------|
| 上一条（←） | 加载上一条待标注任务 |
| 下一条（→） | 加载下一条待标注任务 |
| 提交并下一条 | 提交当前标注结果 → 自动跳下一条 |
| 最后一条标注完 | 提示"全部完成，太棒了！"，弹窗关闭回到列表 |

> 快捷键 ← → 切换任务，ESC 关闭弹窗（已有）。

## 6. 错误原因标签管理

### 6.1 位置

系统配置 → 数据源配置 → 错误原因标签管理

### 6.2 功能

| 功能 | 说明 |
|------|------|
| 列表展示 | 所有错误原因标签（名称/说明/启用状态） |
| 新增/编辑/删除 | 如标签已被引用则禁止删除 |
| 启用/禁用 | 控制标注页下拉框是否出现 |

### 6.3 与违规原因分类的区分

| 模块 | 用途 | 使用者 |
|------|------|-------|
| 违规原因 CASE WHEN | AI模型违规原因分类，看板统计用 | 系统自动 |
| 错误原因标签管理 | 标注员发现错误时选择的错误分类 | 标注员手动 |

## 7. 数据存储

Annotation 表（models.py 已有）：

| 字段 | 说明 |
|------|------|
| raw_data_id | 关联 RawData |
| annotator_id | 标注员 ID |
| result | correct / error / ignore |
| error_tag | 错误原因标签（result=error 时必填） |
| note | 备注（选填） |
| is_submitted | 是否已提交 |
| created_at | 标注时间 |
