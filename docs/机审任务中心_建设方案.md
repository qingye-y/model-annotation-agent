## 一、项目背景与目标

### 1.1 背景

商品审核大模型质检标注工作台需要从 iData 数据平台定期拉取机审数据（AI 审核记录），供标注员进行质检标注。原有方案依赖人工 SQL 查询导出，效率低、无法自动化、不支持分层抽样。

### 1.2 目标

| 目标 | 描述 | 优先级 |
|------|------|--------|
| **自动化取数** | 定时或手动从 iData 拉取数据，持久化到本地数据库 | P0 |
| **分层抽样** | 按线上合规/违规比例等比抽样，保证样本代表性 | P0 |
| **数据可视化** | 看板实时展示违规率、Top 原因、不一致率等 | P0 |
| **SQL 可配置化** | 运维人员可在线修改取数 SQL，无需代码改动 | P1 |
| **批次管理** | 支持批次创建、查看、明细导出、中止、删除 | P1 |
| **双模型互检** | Model A + Model B 并发互检，标记不一致样本 | P2 |

## 四、核心模块建设方案

### 4.1 取数服务（fetch_service.py）

#### 4.1.1 分层抽样算法

**问题**：如果简单随机抽样，样本中违规率可能与线上全量偏差很大。

**方案**：分层抽样，按合规/违规分层，按相同比例从各层抽取。

```
输入：
  - 线上总数 total = 10000
  - 线上合规数 compliant = 8000
  - 线上违规数 non_compliant = 2000
  - 抽样比例 pct = 5%

处理：
  1. 合规抽样数 = 8000 × 5% = 400
  2. 违规抽样数 = 2000 × 5% = 100
  3. 样本总数 = 500，样本违规率 = 100/500 = 20%（与线上完全一致）

实现细节：
  - 使用 MD5(审核id) 作为排序种子，保证排序结果固定（可复现）
  - ROW_NUMBER() OVER (ORDER BY MD5(审核id)) 生成序号
  - Python 端取指定位置的序号（IN positions）
  - OVERSAMPLE = 1.5x 补偿 LEFT JOIN 行倍增
```

#### 4.1.2 LEFT JOIN 行倍增问题

**问题**：detail_sql 包含 `LEFT JOIN dwd_itm_audit_reject_detail_y d`，当一个 app_id 对应多条驳回详情时，LEFT JOIN 产生重复行。

**方案**：
1. COUNT 查询改用 `COUNT(DISTINCT 审核id)` 而非 `COUNT(*)`
2. Python 端按审核id去重（seen_ids set）
3. 违规原因聚合内层先 `GROUP BY 审核id, 创建日期, violation_tag`，外层 `SUM(cnt)`

#### 4.1.3 iData API 调用

```python
def execute_sql_query(sql, instance, env):
    cookie = get_idata_cookie()  # 优先从数据库读取，fallback config.py
    headers = {'Content-Type': 'application/json', 'Cookie': cookie}
    payload = {
        'sql': sql,

        'datasourceType': IDATA_DATASOURCE_TYPE
    }
    resp = requests.post(query_api_url, json=payload,
                         headers=headers, timeout=300)
    result = resp.json()
    return result['data']['values']
```

```python
def fix_encoding(raw_str):
    """latin1 编码的 UTF-8 字节序列 → 正常字符串"""
    return raw_str.encode('latin1').decode('utf-8')

def parse_row(raw_row):
    """解析一行数据，自动处理编码"""
    row = {}
    for col, val in zip(columns, raw_row):
        if col == 'reject_reason' or col == 'AI拒绝原因':
            if val:
                val = fix_encoding(val)
        row[col] = val
    return row
```

### 4.3 SQL 配置模块（sql_config.py）

#### 4.3.1 参数扫描与替换

```python
# 自动扫描 ${参数名} 占位符
def scan_params(sql_text):
    pattern = r'\$\{([^}]+)\}'
    return list(set(re.findall(pattern, sql_text)))

# 替换参数
def replace_params(sql, params_dict):
    for key, value in params_dict.items():
        sql = sql.replace(f'${{{key}}}', str(value))
    return sql
```

#### 4.3.2 取数管道执行

```python
def execute_pipeline(env, instance, start_date, end_date):
    """按管道顺序执行各步骤 SQL"""
    pipeline = FetchPipeline.query.filter_by(env=env, enabled=True)\
        .order_by(FetchPipeline.sort_order).all()

    results = {}
    for step in pipeline:
        template = SqlTemplate.query.get(step.sql_template_id)
        if not template:
            continue

        # 替换参数
        sql = replace_params(template.sql_text, {
            'start_date': start_date,
            'end_date': end_date,
            'instance': instance,
            'year': start_date[:4]
        })

        # 执行
        values = execute_sql_query(sql, instance, env)
        results[step.step_name] = values

    return results
```

## 七、验收标准

### 7.1 取数功能

| 验收项 | 标准 |
|--------|------|
| ZJWC 20260518 COUNT | 5588 = 合规 1109 + 违规 4479（完全对齐 iData 基准） |
| 分层抽样比例 | 样本违规率 = 线上违规率（误差 < 0.1%） |
| MD5 可复现 | 相同参数两次拉取，RawData 记录完全一致 |
| LEFT JOIN 去重 | COUNT(DISTINCT 审核id) 与实际行数一致 |
| DailyStats 准确性 | 按日统计值与 iData 线上 COUNT 对齐（误差 < 1%） |

### 7.2 看板功能

| 验收项 | 标准 |
|--------|------|
| 违规率计算 | violation_rate = non_compliant_count / total_count × 100 |
| Top10 分组 | 超过10种原因时，第11种及以后归入"其他" |
| 不一致率 | 仅统计 review_status='completed' 的批次 |
| 日期筛选 | 仅显示 start_date ≤ stat_date ≤ end_date 范围内的数据 |

### 7.3 SQL 配置功能

| 验收项 | 标准 |
|--------|------|
| 参数扫描 | 修改 SQL 后自动提取新的 ${参数} 占位符 |
| 测试执行 | 点击测试后 5 分钟内返回结果或超时提示 |
| 幂等初始化 | 重复启动不会重复插入 SQL 模板 |

### 7.4 数据导出功能

| 验收项 | 标准 |
|--------|------|
| raw 导出 | 包含该批次全部 RawData，28 个字段 |
| violation 导出 | 仅 ai_result='违规' 的记录 |
| review 导出 | 仅 modelb_reviewed=True 的记录，含模型 B 结果字段 |
| 流式处理 | 超过 500 条时不因内存溢出崩溃 |
