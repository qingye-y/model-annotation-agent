# 标注员效能看板 PRD v1.0

**日期**：2026-05-25

## 1. 背景

当前看板的"今日标注员工作量"模块只展示了标注量的排名，缺少质量维度。管理员无法从看板上快速判断：谁标注得多但准确率低？谁标注得少但质量高？需要将标注量和准确率结合展示，形成完整的效能视图。

## 2. 需求描述

将原"👨‍💻 今日标注员工作量"模块升级为"📊 标注员效能概览"：

| 改动项 | 说明 |
|--------|------|
| 模块名称 | 改为"📊 标注员效能概览" |
| 展示内容 | 标注量 + 标注准确率，双维度展示 |
| 默认排序 | 按准确率从低到高排序（优先关注需要改进的标注员） |
| 可视化 | 横向柱状图展示标注量，右侧或悬停显示准确率 |
| 点击交互 | 点击某个标注员，跳转到任务调度中心并自动打开该标注员的详情弹窗 |

## 3. 卡片设计

每个标注员一个卡片/行，展示：

| 信息 | 展示方式 |
|------|----------|
| 标注员姓名 | 文字 |
| 今日标注量 | 数字 + 横向进度条（相对占比） |
| 标注准确率 | 百分比 + 颜色标识（≥90% 绿色，80-90% 黄色，<80% 红色） |
| 排名变化 | 可选，后续迭代 |

## 4. 排序逻辑

| 排序方式 | 说明 |
|----------|------|
| 默认排序 | 按准确率从低到高（让需要关注的人排前面） |
| 手动切换 | 支持切换为按标注量排序、按准确率排序 |

**为什么默认按准确率从低到高？**

管理员最需要优先看到的是"谁可能有问题"，而不是"谁干得最多"。准确率低的人排在前面，管理员可以第一时间发现并查看详情。

## 5. 与调度中心联动

| 触发 | 行为 |
|------|------|
| 点击看板上的标注员卡片 | 跳转到任务调度中心 |
| 跳转后 | 自动打开该标注员的详情弹窗（通过 URL 参数传递标注员 ID 和弹窗触发指令） |

实现方式：看板点击 → `parent.navigateTo('dispatch_center.html?open_annotator=青也')` → 调度中心页面加载后检测 URL 参数 → 自动调用弹窗。

## 6. 数据来源

| 数据 | 来源 | 接口 |
|------|------|------|
| 今日标注量 | 当天标注提交数 | GET /api/dispatch/annotator-load |
| 标注准确率 | 标注员自评准确率（标注为"正确"的占比） | GET /api/dispatch/annotator-stats?days=1 |
| 最终准确率 | 与质检结果对比 | 🔲 待质检中心完成后替换 |

## 7. 改动范围

| 文件 | 改动 |
|------|------|
| templates/dashboard.html | 模块名称、展示逻辑、排序逻辑、点击跳转 |
| templates/dispatch_center.html | 支持接收 URL 参数自动打开弹窗 |
| blueprints/dashboard.py | 看板接口增加标注员准确率数据 |
| blueprints/dispatch.py | 已有接口，确认数据完整 |
