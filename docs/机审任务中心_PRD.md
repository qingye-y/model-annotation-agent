## 一、产品概述

### 1.1 产品定位

机审任务中心是商品审核大模型质检标注工作台的核心模块，为标注团队提供从 iData 数据平台自动获取机审数据（AI 审核记录）的能力，并支撑数据可视化看板和 SQL 配置管理功能。

### 1.2 目标用户

| 用户角色 | 使用场景 |
|----------|----------|
| **标注员** | 查看批次数据，进行质检标注 |
| **运营人员** | 拉取数据、查看看板、导出报表 |
| **运维人员** | 配置 SQL 模板、管理 Cookie |
| **管理员** | 全量管理（包含以上所有权限） |

### 1.3 核心价值

- **效率提升**：从人工 SQL 导出 → 全自动定时/手动拉取
- **数据一致性**：分层抽样保证样本代表性
- **实时可见**：看板实时展示违规率、Top 原因、不一致率
- **灵活配置**：SQL 模板在线修改，无需代码发布

## 三、详细功能说明

### 3.1 F1 线上数据拉取

#### F1.1 环境与实例选择

**需求描述**：用户选择从哪个环境和哪些实例拉取数据。

**交互要求**：
- 两组单选按钮：云环境 / 乐采云环境
- 选择后，下方实例多选框动态更新
- 云环境可选：浙江网超(ZJWC)、浙江乐采网超(HWCS)、湖南乐采网超(HNLCWC)
- 乐采云环境可选：云南乐采云(YNLCY)、广西乐采云(GXLCY)
- 支持全选/取消全选

**数据来源**：`config.py` 中的 `ENV_CONFIG` 常量

---

#### F1.2 日期范围选择

**需求描述**：指定拉取数据的时间范围。

**交互要求**：
- 两个日期选择器：开始日期、结束日期
- 默认值：开始日期=7天前，结束日期=昨天
- 校验：开始日期 ≤ 结束日期
- 支持快捷选项：近7天、近30天、本月

---

#### F1.3 抽样比例配置

**需求描述**：从线上全量数据中按比例抽取样本。

**交互要求**：
- 数字输入框 + 滑块，范围 1-100%
- 默认值：5%
- 下方显示提示文案：「将从线上全量数据中按相同比例抽取合规和违规样本」
- 抽样比例说明：「样本数 = 线上该类数据 × 抽样比例%。如线上合规 8000 条，违规 2000 条，抽样 5%，则合规样本 400 条，违规样本 100 条，样本总量 500 条」

**业务规则**：
- 分层抽样：合规和违规各自按比例抽取，样本中违规率与线上全量完全一致
- 可复现：相同参数两次拉取结果完全一致（MD5 固定种子）

---

#### F1.4 执行取数

**需求描述**：用户点击「立即拉取」，触发后端取数流程。

**交互要求**：
- 点击后按钮变为「拉取中...」，禁止重复点击
- 下方显示进度：「正在拉取 ZJWC... (1/3)」+ 进度条
- 每完成一个实例，进度更新一次
- 完成后显示结果摘要卡片

**异常处理**：
- Cookie 过期 → 弹窗提示「Cookie 已过期，请在系统配置页更新」
- 网络超时 → 显示错误信息，提供重试按钮
- 部分失败 → 完成的实例正常显示，失败的实例标红并显示原因

---

#### F1.5 取数结果展示

**需求描述**：取数完成后展示结果摘要。

**展示字段**：

| 字段 | 说明 |
|------|------|
| 批次号 | 自动生成，如 BAT-20260525103456 |
| 线上总数 | 该日期范围+实例在 iData 的全量审核数 |
| 合规数 | 线上合规审核数 |
| 违规数 | 线上违规审核数 |
| 本次拉取 | 实际抽取并存储的条数 |
| 数据覆盖日期 | 开始日期 ~ 结束日期 |
| 创建时间 | 本次拉取的时间 |

**操作按钮**：
- 「查看详情」→ 跳转批次详情页
- 「查看看板」→ 跳转数据看板（含该批次数据）

#### F2.3 批次导出

**需求描述**：支持按类型导出批次数据。

**导出类型**：

| type 参数 | 内容 | 字段数 |
|-----------|------|--------|
| `raw` | 原始全量数据（所有记录）| 28 字段 |
| `violation` | 仅违规数据（ai_result=违规）| 28 字段 |
| `review` | 互检差异数据（modelb_reviewed=True）| 31 字段（含 Model B 结果）|

**导出格式**：CSV，UTF-8 BOM 编码（Excel 兼容中文）

**特殊字段处理**：
- 图片 URL：直接输出完整 URL
- 多行文本：双引号包裹，换行符保留
- 空值：输出空字符串（不是 NULL）

## 四、非功能需求

### 4.1 性能要求

| 指标 | 要求 |
|------|------|
| 取数接口响应 | < 30s（不含 iData 查询时间）|
| 看板接口响应 | < 2s（从 DailyStats 读取）|
| 批次明细分页 | < 1s（500 条/批）|
| CSV 导出 | 流式处理，支持 > 10 万条记录 |
| 列表分页 | 每页 20/50 条，前端分页 |

### 4.2 可用性要求

- Cookie 过期自动检测，失败时返回明确错误信息
- iData 查询超时（5 分钟）自动终止
- 数据库连接异常自动重试（3次）
- 取数中断（用户中止/网络中断）后，已拉取的实例数据保留

### 4.3 安全要求

- 所有 API 需登录认证（未登录返回 401）
- SQL 模板不支持多语句（禁止 `;` 后跟新语句）
- Cookie 在前端显示时做掩码处理
- 导出文件不含密码/Token 等敏感信息

### 4.4 数据准确性要求

- COUNT(DISTINCT 审核id) 替代 COUNT(*) 防止 LEFT JOIN 行倍增
- DailyStats 按日期+实例去重（UNIQUE 约束）
- iData 线上全量数与 DailyStats 总数误差 < 1%
- 分层抽样样本违规率与线上全量违规率误差 < 0.1%
