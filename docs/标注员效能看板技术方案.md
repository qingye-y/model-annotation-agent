## 一、后端改动

### 1. blueprints/dashboard.py — 看板接口增加标注员准确率数据

在 `/api/dashboard/stats` 返回中新增 `annotator_stats` 字段：

```json
{
  "annotator_stats": [
    {
      "name": "青也",
      "today_annotated": 10,
      "today_accuracy": 80.0
    }
  ]
}
```

**数据来源**：联表查询 annotation 表当天记录，按 user_id 聚合，计算 correct_count / total_count。

**准确率计算**：
```sql
SUM(CASE WHEN result='correct' THEN 1 ELSE 0 END) / COUNT(*)
```

**返回格式**：`[{name, today_annotated, today_accuracy}, ...]`

### 2. blueprints/dispatch.py — 确认 annotator-load 接口数据完整

`/api/dispatch/annotator-load` 已有返回字段：
- `id`：标注员 ID
- `name`：标注员姓名
- `today_assigned`：今日已分配量
- `today_completed`：今日已完成量

新增字段：
- `today_accuracy`：今日标注准确率（新增）

## 二、前端改动

### 1. templates/dashboard.html

#### 模块改造

1. **模块标题**：`👨‍💻 今日标注员工作量` → `📊 标注员效能概览`
2. **Badge**：`TOP 10` → `今日`
3. **展示方式**：改为横向柱状图（纯 HTML/CSS 渲染，不依赖 ApexCharts）
4. **数据来源**：`/api/dispatch/annotator-load` + `/api/dashboard/stats` 返回的 `annotator_stats`

#### 横向柱状图设计

```
青也  ████████████████████░░░░░░░  45条   78.5% ⚠️
小明  ██████████████████░░░░░░░░░  38条   92.1% ✓
...
```

- 每行：标注员姓名 + 横向进度条 + 标注量数字 + 准确率百分比 + 颜色标识
- 进度条宽度 = 该标注员标注量 / MAX(标注量) * 100%
- 准确率颜色：<80% 红色，80-90% 黄色，≥90% 绿色

#### 排序逻辑

- **默认排序**：按 `today_accuracy` 从低到高
- **排序切换**：两个切换按钮，按准确率排序 / 按标注量排序

#### 点击跳转

- 点击标注员行 → `parent.navigateTo('dispatch_center.html?open_annotator=青也')`
- 调度中心收到参数后自动打开该标注员的详情弹窗

### 2. templates/dispatch_center.html

#### URL 参数支持

页面初始化时（`DOMContentLoaded` 或 `window.initPage`）：

```javascript
// 读取 URL 参数
const urlParams = new URLSearchParams(window.location.search);
const openAnnotator = urlParams.get('open_annotator');

if (openAnnotator) {
  // 找到对应的标注员负载卡片
  const card = findAnnotatorCardByName(openAnnotator);
  if (card) {
    card.click();  // 触发原有的打开详情弹窗逻辑
  }
}
```

URL 参数格式：`dispatch_center.html?open_annotator=青也`
