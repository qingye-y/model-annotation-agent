## 2. 数据分析

### 2.1 现有字段确认

| 字段 | 类型 | 说明 |
|------|------|------|
| `RawData.modelb_consistent` | Boolean | true=一致，false=不一致，null=漏审 |

## 3. 后端实现

### 3.1 新增 API：计算准确率聚合数据

**路由**：`GET /api/dashboard/accuracy-stats`

**查询参数**：

| 参数 | 类型 | 说明 | 默认值 |
|------|------|------|--------|
| `start_date` | string | 开始日期（YYYYMMDD） | 最近7天 |
| `end_date` | string | 结束日期（YYYYMMDD） | 今天 |
| `instances` | string | 实例列表（逗号分隔） | 全部 |

**SQLAlchemy 查询逻辑**：

```python
# 仅纳入已完成互检且模型B有结果的记录
query = RawData.query.filter(
    RawData.modelb_reviewed == True,
    RawData.modelb_consistent != None  # 排除 null（漏审）
)

# 人工标注为 ignore 的不计入
query = query.filter(
    or_(RawData.check_result != 'ignore', RawData.check_result == None)
)
```

**准确率判定函数**：

```python
def is_modela_correct(record):
    """
    判定模型A是否正确。

    规则：
    - modelb_consistent=True（一致）：
        - check_result == 'error' → 模型A错误
        - 否则 → 模型A正确
    - modelb_consistent=False（不一致）：
        - check_result == 'correct' → 模型A正确
        - 否则 → 模型A错误
    """
    if record.modelb_consistent is True:
        # 一致：默认正确，标注为 error 时修正为错误
        return record.check_result != 'error'
    elif record.modelb_consistent is False:
        # 不一致：默认错误，标注为 correct 时修正为正确
        return record.check_result == 'correct'
    else:
        # null 漏审：暂不计入（调用方过滤）
        return None
```

**返回格式**：

```json
{
  "accuracy_rate": 87.5,
  "correct_count": 1234,
  "total_reviewed": 1410,
  "ignored_count": 50,
  "skipped_count": 30,
  "start_date": "20260520",
  "end_date": "20260526",
  "empty": false
}
```

| 字段 | 说明 |
|------|------|
| `accuracy_rate` | 准确率百分比，保留1位小数，无数据时 null |
| `correct_count` | 模型A正确记录数 |
| `total_reviewed` | 参与互检的有效记录数（排除 ignore） |
| `ignored_count` | 被标注为 ignore 的记录数（不参与准确率计算） |
| `skipped_count` | 漏审（modelb_consistent=null）记录数（不参与计算） |
| `empty` | true=无有效数据 |

## 4. 前端实现

### 4.1 准确率卡片

**文件**：`templates/dashboard.html`

**新增函数**：

```javascript
async function loadAccuracyStats() {
  const params = buildDateParams(); // start_date, end_date, instances
  const res = await apiGet('/api/dashboard/accuracy-stats?' + params);
  const card = document.getElementById('accuracyCard');

  if (res.empty || res.accuracy_rate === null) {
    card.innerHTML = '<span class="no-data">暂无互检数据</span>';
    return;
  }

  card.innerHTML = `
    <div class="metric-value">${res.accuracy_rate.toFixed(1)}%</div>
    <div class="metric-sub">正确 ${res.correct_count.toLocaleString()} / 总互检 ${res.total_reviewed.toLocaleString()}</div>
  `;
}
```

**样式**：复用现有 KPI 卡片样式，数字加粗显示。

### 4.2 准确率趋势图

**位置**：与违规率趋势图并列（或上下排列），使用 ApexCharts 实现。

**新增图表配置**：

```javascript
async function loadAccuracyTrend(period) {
  const days = period === '7d' ? 7 : 30;
  const res = await apiGet(`/api/dashboard/accuracy-trend?days=${days}`);

  if (res.empty || !res.trend.length) return;

  const categories = res.trend.map(d => d.date);
  const seriesData = res.trend.map(d => d.accuracy_rate);

  new ApexCharts(document.querySelector('#accuracyTrendChart'), {
    chart: { type: 'line', height: 220 },
    series: [{ name: '准确率', data: seriesData }],
    xaxis: { categories },
    yaxis: {
      max: 100,
      labels: { formatter: v => v.toFixed(0) + '%' }
    },
    stroke: { curve: 'smooth', width: 2 },
    colors: ['#4f8af7'],
    markers: { size: 4 },
    tooltip: {
      custom: function({ dataPointIndex }) {
        const d = res.trend[dataPointIndex];
        return `<div style="padding:8px">
          <div>${d.date}</div>
          <div>准确率：${d.accuracy_rate.toFixed(1)}%</div>
          <div>正确 ${d.correct_count} / 总互检 ${d.total_reviewed}</div>
        </div>`;
      }
    }
  }).render();
}
```

**切换逻辑**：复用"近7天/近30天"切换按钮，调用 `loadAccuracyTrend()` 重新请求并渲染。

### 4.3 筛选联动

准确率卡片和趋势图随日期/实例筛选联动：

- `buildDateParams()` 函数获取当前筛选器状态（`start_date`、`end_date`、`instances`）
- 准确率卡片和趋势图共用同一参数构建逻辑
- 实例筛选变化时，重新请求两个 API

## 6. 验收标准

| # | 验收条件 | 验证方式 |
|---|---------|---------|
| V1 | 无互检数据时，卡片显示"暂无互检数据" | DB 清空互检数据后刷新看板 |
| V2 | 完成互检后（无人工标注），卡片按默认规则显示准确率 | DB 插入 `modelb_consistent=True/False` 记录，验证计算结果 |
| V3 | 人工标注某条记录后，刷新看板，准确率随之变化 | 标注列表中标注记录，刷新看板对比 |
| V4 | 标注从正确→错误，准确率下降 | 反向标注验证 |
| V5 | 标注从错误→正确，准确率上升 | 正向标注验证 |
| V6 | 趋势图按天展示准确率 | 切换到有多天数据的日期范围 |
| V7 | 趋势图支持近7天/30天切换 | 点击切换按钮，观察 X轴变化 |
| V8 | 按实例筛选时，卡片和趋势图只反映选中实例 | 单实例筛选，验证两个图表数值一致性 |
| V9 | ignore 记录不计入准确率分母 | 标注一条为 ignore，验证分母不变 |

## 7. 风险与注意事项

| 风险 | 影响 | 缓解 |
|------|------|------|
| 历史数据无 `modelb_consistent` 值 | 准确率分母为0 | API 需正确返回 `empty=true` |
| `gmt_created` 字段为空 | 按天分组失败 | 查询时过滤 `gmt_created IS NOT NULL` |
| `check_result` 为 None vs 空字符串 | 逻辑判断需严格区分 `!= 'error'` vs `is not None` | Python 中 `None != 'error'` 为 True，需明确处理 |
