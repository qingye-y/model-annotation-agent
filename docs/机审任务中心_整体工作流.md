> 版本：v1.0.0 | 日期：2026-05-25 | 作者：青也 / 小B

## 一、系统定位与目标

机审任务中心是商品审核大模型质检标注工作台的核心模块，承载以下职责：

1. **从 iData 数据平台自动拉取机审数据**（AI 审核记录）
2. **按分层抽样策略**抽取样本，持久化存储于本地 SQLite 数据库
3. **生成每日统计快照**，供看板页面实时展示
4. **支持批次管理**（创建、查看、导出、删除）
5. **支持 SQL 配置化**，运维人员可在线修改取数 SQL

## 三、详细工作流

### 3.1 取数工作流（核心链路）

```
用户操作                          后端处理                            数据平台
  │                                 │                                  │

  │   POST /api/data-fetch          │                                  │

  │                                 │                                  │

  │                                 │    "BAT-20260525103456"          │
  │                                 │                                  │
  │                                 │ ③ 写入 FetchLog                  │
  │                                 │    status = "running"            │
  │                                 │                                  │
  │                                 │ ④ 遍历实例列表                    │
  │                                 │    for instance in [ZJWC, HWCS]: │
  │                                 │                                  │
  │                                 │ ⑤ ping_idata()                  │
  │                                 │    Cookie有效性探活               │
  │                                 │────────────────────────────────►│
  │                                 │◄─────────────────────────────────│
  │                                 │    OK / Cookie过期               │
  │                                 │                                  │
  │                                 │ ⑥ 构建 COUNT SQL                 │
  │                                 │    查询该实例+日期范围的          │
  │                                 │    线上总数/合规数/违规数         │
  │                                 │────────────────────────────────►│
  │                                 │    sql: SELECT ... COUNT(*)      │
  │                                 │◄─────────────────────────────────│
  │                                 │    {values: [[5588], [1109],...]}│
  │                                 │                                  │
  │                                 │ ⑦ 分层抽样计算                    │
  │                                 │    合规样本 = 1109 × 5% = 55     │
  │                                 │    违规样本 = 4479 × 5% = 224    │
  │                                 │    MD5 固定种子可复现             │
  │                                 │                                  │
  │                                 │ ⑧ 执行抽样 SQL（窗口函数）        │
  │                                 │    ROW_NUMBER() OVER MD5排序    │
  │                                 │────────────────────────────────►│
  │                                 │◄─────────────────────────────────│
  │                                 │    values: [[商品1], [商品2],...]│

  │                                 │ ⑨ Python 端去重截断              │
  │                                 │    seen_ids 去重                │
  │                                 │    分类截断（合规55/违规224）    │
  │                                 │                                  │
  │                                 │ ⑩ 写入 RawData 表               │
  │                                 │    fetch_batch_id = batch_id    │
  │                                 │    source = "fetch"              │
  │                                 │    28 个字段完整映射             │
  │                                 │                                  │
  │                                 │ ⑪ 拉取违规原因（线上全量）        │
  │                                 │    fetch_error_reasons_online() │
  │                                 │    CASE WHEN 关键词→标签映射     │
  │                                 │────────────────────────────────►│
  │                                 │◄─────────────────────────────────│
  │                                 │    {水印:500, 类目错放:300, ...} │
  │                                 │                                  │
  │                                 │ ⑫ generate_daily_stats()        │
  │                                 │    · 按日期+实例聚合             │
  │                                 │    · 优先用 iData 精确计数       │

  │                                 │                                  │
```

## 四、数据存储位置

### 4.1 SQLite 数据库（app.db）

| 表名 | 存储内容 | 数据来源 | 生命周期 |
|------|----------|----------|----------|
| **FetchLog** | 取数批次日志（批次ID/总数/状态/时间） | 取数接口自动写入 | 长期保留，手动删除 |
| **RawData** | 原始数据（商品/审核结果/图片URL等28字段） | 取数接口自动写入 | 同批次，清理批次时级联删除 |
| **DailyStats** | 每日快照（按日期+实例聚合） | 取数接口自动写入 | 长期保留，按日期+实例去重 |
| **SqlTemplate** | SQL 模板（可配置CRUD） | 运维手动配置 + 初始化预置 | 手动管理 |
| **FetchPipeline** | 管道执行顺序配置 | 初始化预置 | 手动管理 |
| **User** | 用户账户 | 手动管理 | 长期保留 |
| **Annotation** | 标注记录（Model B 人审结果） | 标注员操作写入 | 长期保留 |
| **QcRecord** | 质检记录 | 质检员操作写入 | 长期保留 |
| **DispatchLog** | 任务分发记录 | 分发中心写入 | 长期保留 |
| **AuditLog** | 操作审计日志 | 各操作自动记录 | 日志表 |

### 4.3 DailyStats 字段与数据来源映射

```
DailyStats 字段           数据来源
────────────────────────────────────────────
stat_date                 ←  取数日期范围，按日分组
instance_code             ←  遍历实例
total_count              ←  优先：iData COUNT(DISTINCT 审核id) 精确值

```

## 五、iData API 调用规范

### 5.1 API 端点

| 环境 | 端点 URL |
|------|----------|
| 云环境（ZJWC/HWCS/HNLCWC） | `https://idata.cai-inc.com/api/idas/inner/fetchData/getData` |
| 乐采云环境（YNLCY/GXLCY） | `https://idata.cai-inc.com/lcy_idas/api/idas/inner/fetchData/getData` |

### 5.2 请求格式

```http
POST {api_url}
Content-Type: application/json
Cookie: {IDATA_COOKIE}

{
  "sql": "SELECT ... FROM dwd.dwd_itm_audit_app_ai_result_detail_y ...",
  "instance": "ZJWC",
  "datasourceType": "hive"
}
```

### 5.3 响应格式

```json
{
  "success": true,
  "data": {
    "count": 5588,
    "columns": ["字段1", "字段2", ...],
    "values": [
      ["值1", "值2", ...],
      ["值1", "值2", ...]
    ]
  }
}
```

### 5.4 Cookie 管理

- **存储位置**：SqlConfig 表（key='IDATA_COOKIE'）
- **获取方式**：浏览器登录 iData 后，F12 开发者工具 → Network → 任意请求 → Request Headers → Cookie
- **有效期**：通常 1-7 天，过期后取数接口会报 401，需重新更新
- **配置路径**：系统配置页（sql_config.html）→ Cookie 管理 Tab

### 5.5 编码规范

HTTP Body 是 UTF-8 JSON，但 `reject_reason`（AI拒绝原因）字段内部是 latin1 编码的 UTF-8 字节序列，需要两层解码：

```python
# 第一层：JSON 本身 UTF-8 解码
raw_str = raw_response['reject_reason']

# 第二层：latin1 → UTF-8
decoded_str = raw_str.encode('latin1').decode('utf-8')
```

## 六、批次生命周期

```
  创建批次
    │
    ├─ FetchLog: status=running
    │
    ├─ RawData: 写入中（每实例 commit 一次）
    │
    └─ 完成后：
         │
         ├─ FetchLog: status=completed
         │              original_total=线上全量数
         │              total_fetched=抽样后条数
         │
         ├─ DailyStats: 按日期+实例写入快照
         │
         └─ RawData: 关联 batch_id，source='fetch'
                        │
                        ▼
                   标注员标注
                        │
                        ├─ Annotation: 写入人审结果
                        │
                        └─ RawData.check_result: 1(正确)/0(错误)
                                   │
                                   ▼
                            模型互检（可选）
                                   │
                                   ├─ RawData.modelb_result/reason/consistent
                                   └─ FetchLog.inconsistent_count 更新
                                              │
                                              ▼
                                         导出数据
                                              │
                                              ├─ api_download(type=raw)  → 原始全量 CSV
                                              ├─ api_download(type=violation) → 违规数据 CSV
                                              └─ api_download(type=review) → 互检差异 CSV
```

## 七、关键参数说明

| 参数 | 含义 | 取值范围 | 默认值 |
|------|------|----------|--------|
| `sample_percent` | 抽样比例（相对于线上全量） | 1-100 | 5 |
| `start_date` | 数据开始日期（YYYYMMDD） | 历史日期 | 7天前 |
| `end_date` | 数据结束日期（YYYYMMDD） | 历史日期 | 昨天 |

| `instances[]` | 实例列表 | ZJWC/HWCS/HNLCWC/YNLCY/GXLCY | 全选 |
| `type`（导出）| 导出类型 | raw/violation/review | raw |

## 八、错误处理

| 错误场景 | 处理方式 | 影响范围 |
|----------|----------|----------|
| Cookie 过期 | ping_idata() 失败，返回 401 错误码 | 全部取数失败 |
| iData 查询超时（>300s） | 抛出异常，FetchLog 标记 status=failed | 当前实例取数中断 |
| 数据量为0 | 跳过写入，日志记录 | 不影响其他实例 |
| 重复 batch_id | 自动生成时间戳保证唯一 | 无 |
| RawData 清理失败 | 忽略（delete 未匹配行是正常的） | 无 |
