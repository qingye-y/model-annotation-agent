| `reject_reason` | AI 拒绝原因 |
| `reject_detail` | AI 拒绝解释 |
| `check_result` | 线上已有的标注结果（1 = 正确，0 = 错误） |
| `annotation` | 线上备注 |

| `audit_status` | 审核单状态（0 = 待审核 / 3 = 审核驳回 / 4 = 审核通过 / 5 = 商家撤回） |
| `audit_idea` | 人审意见 |
| `gmt_created_time` | 创建时间（分区字段 `pt` 基于此年份） |

## 关联逻辑（取数 SQL）

取数 SQL 以 **AI 审核结果表（`r`）** 为主表，通过以下 LEFT JOIN 关联：

```
r（dwd_itm_audit_app_ai_result_detail_inc_y）
  ├── LEFT JOIN d（dwd_itm_audit_reject_detail_y）
  │     ON r.app_id = d.app_id
  │     获取本次审核的驳回项/原因
  │
  ├── LEFT JOIN c（dim_itm_basic_info_detail_d）
  │     ON r.goods_id = c.id AND r.instance_code = c.instance_code
  │     获取商品层面的店铺名称、发布渠道标签
  │
  └── LEFT JOIN new（dwd_itm_audit_detail_inc_y）
        ON new.item_id = r.goods_id
        获取该商品当天的审核类型组合（变更类别）
```

### 查询条件

- `r.pt = '2026'` 且 `d.pt = '2026'` — 按年分区过滤
- `date_format(r.gmt_created_time,'%Y%m%d') between ... and ...` — 按日期过滤
- `r.instance_code in ('GXLCY','YNLCY','HWCS','ZJWC','HNLCWC')` — 限制五个实例
- `audit_status_name <> '已撤回'` — 排除已撤回记录

## 后端取数模块说明

在 `data_fetch.py` 的 `build_sql()` 中：

- SQL 中的 `{start_date}`, `{end_date}`, `{instance}`, `{year}` 会被动态替换为实际参数
- 查询结果集的列名在 `fetch_data_from_idata()` 中被正确映射到 `RawData` 模型的对应字段
